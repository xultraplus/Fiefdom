这是一个非常出色且具有高度差异化竞争力的游戏概念。将《星露谷物语》的种田玩法移植到中国“礼崩乐坏”但充满生机的春秋时期，用**“井田制”**替换**“资本主义农场”**，用**“养士”**替换**“雇佣”**，这在目前的游戏市场上是一个非常独特的切入点（USP）。

为了将你的两个文档整合成一份**专业游戏开发标准**的文档，我们需要遵循工业界的标准流程：**概念设计（Concept） -> 宏观设计文档（GDD） -> 垂直切片（Vertical Slice）**。

以下是融合了你的两份文件，并按照**专业游戏策划案（GDD - Game Design Document）**标准重构的完整方案。此外，我还在最后补充了专业团队的开发流程建议。

# 《采邑春秋》（Project: Fiefdom）核心设计文档

**版本**：1.0
**类型**：模拟经营 / 角色扮演 / 历史策略
**视角**：2D 俯视角 / 像素或手绘风
**对标作品**：《星露谷物语》、《守墓人》、《大周列国志》

---

## 1. 游戏愿景 (High Concept)

**一句话介绍**：
一款以中国春秋时期为背景的“士大夫模拟器”。玩家扮演没落贵族，在礼崩乐坏的乱世中经营封地（采邑），通过遵循“井田制”复兴经济，修习“君子六艺”提升阶层，招募诸子百家为门客，最终在霸主争雄的时代中守护一方桃源。

**核心体验支柱**：
1.  **考究的国风种田**：不仅是种菜，更是对“五谷、桑蚕、井田”的系统模拟。
2.  **士大夫的阶层跃迁**：从生存到复礼，从“士”到“卿”，体验等级森严的贵族生活。
3.  **门客管理与自动化**：如同孟尝君般养士，将繁琐劳动转化为人员管理策略。

---

## 2. 核心循环 (Core Loop)

专业设计通常将游戏分为三层循环：

*   **秒级循环（操作）**：移动、挥舞青铜农具、交互、战斗。
*   **日级循环（战术）**：
    *   **卯时（规划）**：查看黄历/政令，分配门客任务。
    *   **辰-申时（执行）**：亲耕私田（恢复体力）、督办公田（维持爵位）、外出探索（攘夷/采集）。
    *   **酉-亥时（修养）**：夜读修习六艺、社交宴请、处理突发事件。
*   **长线循环（战略）**：
    *   季度（节气）：播种 -> 收获 -> 祭祀 -> 纳贡。
    *   年度：爵位晋升 -> 解锁新科技（铁器/牛耕） -> 扩大封地 -> 家族联姻。

---

## 3. 系统机制详解 (Game Mechanics)

### 3.1 经济系统：井田制的博弈
这是本作的核心差异点。地图地块默认为 **3x3 九宫格**。

*   **公田（中央格）**：
    *   **机制**：产出强制上缴国君。玩家无法获取收益，但**必须**投入劳动力。
    *   **奖惩**：公田维护好 -> 获得“国君赏赐”（稀有蓝图、青铜礼器）；维护差 -> 扣除“声望”，甚至导致**“削爵”**（Game Over风险）。
*   **私田（周边八格）**：
    *   **机制**：产出归玩家。
    *   **作物**：五谷（黍、稷、麦、菽、稻）。
    *   **桑蚕产业链**：种桑 -> 养蚕（温控小游戏） -> 缫丝 -> 织绸。丝绸是高级货币，用于外交和贿赂。
*   **改革（后期玩法）**：
    *   玩家可选择推行**“初税亩”**（私有化改革）。废除公田，全变私田，收入暴增，但会大幅降低“周礼”评价，导致儒家门客离去，法家门客涌入，外交关系恶化。

### 3.2 角色成长：君子六艺 (The Six Arts)
替代传统的 RPG 属性（力量/敏捷/智力）。

| 技能 | 对应玩法 | 升级解锁内容 |
| :--- | :--- | :--- |
| **礼 (Rites)** | 祭祀、社交、着装规范 | 提升爵位上限（士->大夫），解锁高级外交，避免被霸主讨伐。 |
| **乐 (Music)** | 铸钟、编钟演奏(音游) | 范围Buff（提升门客效率），安抚民心，增加宴会成功率。 |
| **射 (Archery)** | 狩猎、塔防射击 | 提升远程伤害，解锁复合弓、毒箭，获取稀有毛皮。 |
| **御 (Chariot)** | 驾车跑图、战车冲撞 | 提升移动速度，解锁**“战车冲锋”**（对群怪大杀器），增加背包负重。 |
| **书 (Calligraphy)** | 夜读、修复竹简 | 解锁科技树（水利、农具配方），提升辩论（舌战）胜率。 |
| **数 (Math)** | 交易、账目管理 | 降低纳税比例（精算），提升商店售价，优化门客AI逻辑。 |

### 3.3 门客系统 (Retainer System)
替代《星露谷》的祝尼魔（Junimos），结合《环世界》的简化版管理。

*   **招募来源**：流民（消耗粮）、游学士子（消耗声望/书籍）、落魄贵族（消耗丝绸/玉器）。
*   **派系与性格**：
    *   **儒家**：擅长教化（增加治安），厌恶法家政策，喜欢祭祀。
    *   **墨家**：擅长工巧（自动修缮设施、制作机关），吃苦耐劳，厌恶奢靡。
    *   **农家**：擅长种田（产量加成），对政治无感。
    *   **兵家**：擅长战斗（守卫封地），消耗粮食巨大，脾气暴躁。
*   **互动**：若“食宿”待遇太差（如给兵家吃糟糠），门客会**“卷款潜逃”**或**“倒戈”**。

### 3.4 战斗与探索：攘夷 (Expedition)
*   **核心逻辑**：保卫家园（Tower Defense Lite）+ 资源获取（Dungeon Crawler）。
*   **敌人**：
    *   野人/盗匪（初期）：破坏农田，偷窃物资。
    *   戎狄蛮族（中期）：有组织的进攻，需建立烽火台防御。
    *   上古异兽（后期/副本）：夔牛、虎蛟等，产出祭祀用的“神性材料”。
*   **死亡机制**：
    *   **被俘**：战败不读档，而是被蛮族俘虏。需选择“缴纳赎金”或“等待门客救援”。

## 4. 美术与音效 (Art & Audio)

*   **视觉风格**：
    *   **青绿山水 (Blue-Green Landscape)**：参考《千里江山图》，使用石青、石绿、赭石为主色调。
    *   **青铜美学**：UI 采用青铜器铭文、饕餮纹饰；字体采用在此基础上的变体（由于大篆难以辨认，需做艺术化处理）。
*   **听觉体验**：
    *   **金石之声**：UI 点击音效使用玉石撞击声、编钟声。
    *   **环境音**：强调自然的虫鸣鸟叫与古代乐器（埙、琴、筑）的结合。

## 5. 专业开发建议：如何落地？

如果这是一个真实的项目，接下来的开发流程应该如下：

### 第一阶段：原型验证 (Prototyping) - 1个月
*   **目标**：验证“井田制”是否好玩。
*   **做法**：不要做美术，用方块代替。
    *   在 Godot 中快速搭建 3x3 的格子。
    *   实现核心逻辑：中间格子的产出必须上缴，周围格子的产出归自己。
    *   **验证点**：玩家是否会感到“上缴”很痛苦？如果很痛苦，如何通过“赏赐”机制来平衡这种挫败感？

### 第二阶段：垂直切片 (Vertical Slice) - 3~4个月
*   **目标**：制作出“完美的一周”。
*   **内容**：
    *   **美术定调**：画出一套高质量的角色、农作物、地块TileSet。
    *   **功能闭环**：玩家能种植五谷、能养蚕、能去公田劳作、能过一天（日夜循环）、能触发一个节气事件（如春社）。
    *   **技术地基**：建立好 Godot 的 Resource 系统（用于存储成百上千的物品数据）。

### 第三阶段：量产 (Production)
*   **铺量**：根据垂直切片的标准，制作所有的 24 节气事件、所有的 NPC、所有的六艺技能。

### 给 Godot 开发者的具体技术贴士
1.  **数据驱动 (Data-Driven)**：
    *   利用 Godot 的 `Resource` 类。创建一个 `ItemData` 类，包含 `name`, `icon`, `type` (五谷/丝绸), `price`。
    *   创建一个 `RetainerData` 类，包含 `school` (儒/墨/兵), `loyalty`, `skill_level`。
    *   这样你可以在编辑器里直接配表，不需要写死在代码里。

2.  **瓦片地图交互**：
    *   使用 Godot 4.x 的 `TileMapLayer`。
    *   利用 `Custom Data Layers` 给 Tile 加上属性：`is_public_field` (布尔值)。
    *   当玩家点击收获时：
        ```gdscript
        var tile_data = tile_map.get_cell_tile_data(coords)
        if tile_data.get_custom_data("is_public_field"):
            add_to_king_treasury() # 进公仓
        else:
            add_to_player_inventory() # 进背包
        ```

3.  **时间系统**：
    *   不要用 `Timer` 节点做游戏内时间，建议写一个全局 `TimeManager` 单例。
    *   变量：`current_jieqi` (节气), `current_hour` (时辰)。
    *   光照：使用 `CanvasModulate` 节点，根据 `TimeManager` 的时间插值改变颜色（白天白色，晚上深蓝色，黄昏橙红色）。

### 总结
你现在的文档已经具备了**商业独立游戏**的雏形。
*   **亮点**在于文化背景的深度挖掘（井田、六艺、门客）。
*   **风险**在于系统的复杂性（尤其是门客AI和战斗系统）。
*   **建议**：初期大幅简化战斗（甚至可以做成战棋或自动战斗），将资源集中在**“种田+士大夫经营”**这个最核心的体验上。
